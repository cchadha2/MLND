Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer: The benefit to splitting a dataset between training and testing subsets is that it provides the learning algorithm most of the dataset to train itself on (training set) and some of the dataset to validate its predictions upon (testing set). It is also advised to split the data into three subsets: training, cross-validation, and test. As before, the training set is used to train the model. However, the cross-validation set is used here instead to test the predictions of the model and finally, the test set is simply used to make predictions. In terms of the bias-variance tradeoff, splitting the dataset into training and testing sets is highly valuable. If we look at the model complexity graphs or learning curves of a range of models, splitting the dataset can easily show us which models are overfitting and which are underfitting. For example, as we increase the number of training points provided to the models, those that begin with low training error and high cross-validation error will see a gradual convergence into a similar training and CV error (relatively high). This means that the model is underfitting (high bias) and does not accurately reflet the complexity of the data. Similarly, an overfitting (high variance) model will begin with very low training error and very high CV error. Training error will remain quite low and CV error will reduce a small amount but remain high as the number of training points increase. This means that the model has basically memorised the data in the training set and does not generalise well (predictions will be inaccurate on data that is not the training set). However, the optimal model will have low training error and high CV error to begin but CV error will rapidly decrease as the number of training points increase and the two errors converge to relatively low amounts. This model will accurately reflect the complexity of the underlying data and should provide much better predictions than the other models. Without splitting the data, we would not be able to validate the model we create, effectively leaving us blind to whether it is underfitting or overfitting.

Question 7 - Grid Search¶

  • What is the grid search technique?
  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer: The grid search technique is a way of systematically working through multiple combinations of parameter tunes, while cross-validating each parameter combination, to determine which parameters provide the best performance based on a predetermined metric. GridSearchCV is a hyper-parameter optimiser in the sklearn.model_selection module. The user provides GridSearchCV a dictionary of parameters that they are looking to tune (each passed as an array of values including strings) and a metric to score each model using make_scorer from sklearn.metrics and a suitable metric (e.g. f1_score from sklearn.metrics). An object of GridSearchCV is then created using as parameters: the model we are looking to tune (with default parameters set) and the parameters we are looking to tune as well as the scoring metric. This object creates a grid of models, with each possible combination of parameters, trained on the training set. Each model is given a score (based on the metric previously chosen) using the cross-validation set. Finally, the model with the best score is chosen by fitting the object to our training data and using the .bestestimator function. For example, if we were looking to tune max_depth and min_samples_leaf of a decision tree classifier, we would create a dictionary such as: parameters = {'max_depth': [2, 3, 4, 5, 7, 9], 'min_samples_leaf': [1, 2, 4, 6, 8, 10]} and a scorer: scorer = make_scorer(f1_score). An object of the GridSearchCV class would then take the following shape: grid_obj = GridSearchCV(clf, parameters, scoring = scorer) where clf = sklearn.tree.DecisionTreeClassifier(). Note the empty parameters in the classifier. The GridSearchCV object is then fit to the training data: grid_fit = grid_obj.fit(X_train, y_train). This is a grid of decision trees made up of each possible combination of max_depth and min_samples_leaf as we presented in the dictionary, with all other parameters set to default (hence the empty parameters in classifier). From this, we can find our best estimator: best_clf = grid_fit.bestestimator as previously mentioned. Fitting best_clf to our data will give us the best predictions based on our chosen metric (F1 score).

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer: k-fold cross-validation is a method of cross-validating a model 'k' times and then taking the average of the cross-validation results as our model. Instead of simply splitting our dataset into some ratio of training and testing sets, it is split into 'k' sets or folds. The model is trained on the data in k-1 folds and tested on one fold. This is repeated k times with the fold used for testing changing each time. After k repititions, the model will have been tested on each fold once. An average of the results is then taken to build our model. The folds can also be stratified to include approximately the same percentage of samples of each target class as the complete set and can also be shuffled before divided into folds to decrease bias in the data. This technique provides a benefit for grid search when opimising a model as grid search previously hinged on simply using a binary split in our dataset to cross-validate and find optimal hyper-parameters. Essentially, training our model on a portion of data and testing on a smaller portion. This method may not take into account variations in the training and testing sets. However, with k-fold cross-validation, grid search is able to use all of the data to both train and test on which subsequently results in finer tuning of our chosen parameters.

    # Create cross-validation sets from the training data

    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    #cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)
    cv_sets = ShuffleSplit(n_splits=10, test_size=0.2, train_size=0.8, random_state=0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': np.array(range(1,11))}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer'

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
